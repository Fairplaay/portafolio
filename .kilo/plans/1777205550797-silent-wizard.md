# Plan: Internacionalización Estática + Generación de PDF

## Objetivo
Mantener el sitio 100% estático, compatible con Cloudflare Pages, con internacionalización en rutas `/es` y `/en`, y generar los CVs como archivos estáticos pre-construidos.

---

## Fase 1: Reestructuración de i18n

### 1.1 Mover página principal a `/es`
- `src/pages/index.astro` → `src/pages/es/index.astro`
- `src/pages/en/index.astro` → `src/pages/en/index.astro` (ya existe)

### 1.2 Actualizar `astro.config.mjs`
```js
i18n: {
  locales: ['es', 'en'],
  defaultLocale: 'es',
  routing: {
    prefixDefaultLocale: true  // Esto hace que /es sea explícito
  }
}
```

### 1.3 Actualizar componentes de i18n
- `src/i18n/utils.ts`: Ya funciona correctamente extrayendo el lang de la URL
- No se necesita cambios en `ui.ts`

### 1.4 Agregar botón flotante de toggle
En `src/components/Footer.astro`:
- Agregar un botón flotante pequeño en la esquina inferior derecha
- El toggle cambia entre `/es/` y `/en/`
- Estilo: posición fija, z-index alto, pequeño (24x24px o similar)

---

## Fase 2: Script de Generación de PDF

### 2.1 Dependencias
Agregar `pdfmake` al proyecto:
```bash
npm install pdfmake
```

### 2.2 Crear script `scripts/generate-cv.mjs`
El script:
- Lee `src/data/aboutMe.ts` (import dinámico o como módulo)
- Lee `src/data/experience.ts`
- Lee traducciones de `src/i18n/ui.ts`
- Genera dos PDFs: `public/cv-es.pdf` y `public/cv-en.pdf`
- Usa pdfmake para construir el documento con el mismo diseño del CV actual:
  - Sidebar con perfil, contacto, skills, idiomas
  - Contenido principal con About y Experience
  - Colores: #242633 (sidebar), #dbc404 (acentos), #ffffff (fondo)
  - Tipografía: Inter

### 2.3 Estructura del documento pdfmake
```javascript
{
  content: [
    {
      columns: [
        { width: 260, ... sidebar },
        { ... mainContent }
      ]
    }
  ],
  styles: { ... }
}
```

### 2.4 Integrar en package.json
```json
"scripts": {
  "generate:cv": "node scripts/generate-cv.mjs",
  "build": "npm run generate:cv && astro build"
}
```

## Fase 3: Actualizar componentes para PDF estático

### 3.1 `src/components/sections/Hero.astro`
Cambiar:
```astro
const cvDownloadUrl = `/api/cv/${lang}`;
```
Por:
```astro
const cvDownloadUrl = lang === 'es' ? '/cv-es.pdf' : '/cv-en.pdf';
```

### 3.2 Eliminar `src/pages/api/cv/[lang].ts`
Ya no se necesita el endpoint de API. El PDF será servido como archivo estático desde `/public/`.

---

## Fase 4: Verificar compatibilidad con Cloudflare Pages

### 4.1 Configuración de `astro.config.mjs`
```js
output: 'static',
adapter: cloudflare({
  static: true  // Genera output estático compatible con Cloudflare Pages
})
```

### 4.2 Asegurar que no hay prerender false
- Eliminar cualquier `export const prerender = false`
- Todos los endpoints deben ser estáticos
